# Скрипты проекта

Эта папка содержит вспомогательные скрипты для автоматизации различных задач в проекте.

## Доступные скрипты

### `clean-node-modules.sh`
Очищает все каталоги `node_modules` и файлы `package-lock.json` в проекте.

**Использование:**
```bash
./scripts/clean-node-modules.sh
```

**Функциональность:**
- Находит и удаляет все каталоги `node_modules`
- Удаляет все файлы `package-lock.json`
- Выводит отчет о выполненных действиях

### `step-refs.py`
Проверяет соответствие REF меток между кодом и метаданными уроков.

**Использование:**
```bash
./scripts/aliases/step-refs.py           # проверка текущего шага
./scripts/aliases/step-refs.py 02        # проверка шага 02
./scripts/aliases/step-refs.py 02 04     # проверка шагов с 02 по 04 включительно
```

**Функциональность:**
- Извлекает все REF метки из файлов шага
- Извлекает все REF ссылки из meta.yaml для конкретного шага
- Сравнивает множества меток и ссылок
- Выдает отчет с указанием несоответствий
- Используется внутри алиаса `step-copy`

### Добавление новых скриптов

При добавлении новых скриптов:
1. Размещайте их в этой папке
2. Убедитесь, что скрипт имеет права на выполнение (`chmod +x`)
3. Обновите этот файл с описанием нового скрипта
4. Обновите документацию в `docs/guides/aliases.md` при необходимости

## Права доступа

Все скрипты должны быть исполняемыми. Для установки прав используйте:
```bash
chmod +x scripts/имя_скрипта.sh
```

## Совместимость

Скрипты разработаны для работы в Unix-подобных системах (Linux, macOS). Для Windows может потребоваться адаптация или использование WSL.
