# Детальный план занятия - Объектная модель браузера

## Цели и задачи

### Основные цели:
- Познакомить студентов с объектной моделью браузера (DOM)
- Научить находить и модифицировать элементы на странице
- Развить навыки работы с современными DOM API
- Объяснить различия между традиционными и современными методами

### Задачи обучения:
- Уметь получать доступ к элементам страницы
- Уметь изменять стили элементов страницы
- Уметь добавлять и удалять элементы на странице
- Понимать разницу между различными методами поиска элементов
- Уметь работать с классами и data-атрибутами

### Критерии успеха:
- Студент может находить элементы по различным селекторам
- Студент правильно использует методы classList для управления классами
- Студент применяет современные методы (querySelector, remove, append)
- Студент понимает навигацию по DOM (parent, children, closest)
- Студент использует data-атрибуты для хранения пользовательских данных

## Тайминг занятия (90 минут)

### Введение (15 минут)
- **0-5 мин:** Приветствие, организационные моменты, повторение предыдущей темы (массивы)
- **5-10 мин:** Обзор темы занятия, постановка целей, важность DOM в веб-разработке
- **10-15 мин:** Введение в DOM - что такое DOM и зачем он нужен, структура DOM-дерева

### Основная часть (60 минут)

#### Блок 1: Поиск элементов в DOM (20 минут)
- **15-25 мин:** Теория - методы document, querySelector, querySelectorAll, традиционные методы
- **25-30 мин:** Практика - Задачи 1 и 2 (поиск элементов списка)
- **30-35 мин:** Разбор различий между методами поиска, вопросы

#### Блок 2: Навигация и работа с атрибутами (20 минут)
- **35-45 мин:** Теория - children, parentElement, closest, className, classList, dataset
- **45-50 мин:** Практика - Задачи 3 и 4 (навигация по таблице, подсчет элементов)
- **50-55 мин:** Разбор работы с классами и data-атрибутами, вопросы

#### Блок 3: Модификация DOM (20 минут)
- **55-65 мин:** Теория - append, remove, изменение стилей, современные vs устаревшие методы
- **65-70 мин:** Практика - дополнительные примеры модификации DOM
- **70-75 мин:** Разбор интересных фактов, вопросы по модификации DOM

### Заключение (15 минут)
- **75-80 мин:** Повторение ключевых концепций, интересные факты о DOM
- **80-85 мин:** Ответы на вопросы студентов, разбор сложных моментов
- **85-90 мин:** Резюме занятия, домашнее задание, подготовка к следующему уроку

## Критерии оценки

### Формативное оценивание:
- **Активность на занятии** - участие в обсуждениях, задавание вопросов
- **Выполнение практических заданий** - правильность синтаксиса, логика решения
- **Понимание концепций** - способность объяснить разницу между методами DOM
- **Работа в парах/группах** - совместное решение задач, помощь товарищам

### Суммативное оценивание:
- **Правильность выполнения задач** - соответствие кода требованиям задания
- **Понимание методов DOM** - корректное использование querySelector, classList, dataset
- **Применение современных возможностей** - использование современных методов вместо устаревших
- **Читаемость кода** - понятные имена переменных, правильное форматирование

### Контрольные точки:
- Задача 1: корректный поиск элемента списка с использованием querySelector
- Задача 2: правильный поиск всех элементов списка с использованием querySelectorAll
- Задача 3: корректная навигация от td к tr с использованием closest или parentElement
- Задача 4: правильный подсчет дочерних элементов с использованием children.length

## Домашнее задание

### Обязательная часть:
1. Создать HTML-страницу с несколькими элементами и найти их с помощью различных методов
2. Реализовать функцию для переключения класса 'active' у кнопки при клике
3. Создать динамический список, который можно добавлять и удалять элементы
4. Использовать data-атрибуты для хранения дополнительной информации об элементах

### Дополнительная часть:
- Изучить разницу между NodeList и HTMLCollection в деталях
- Попрактиковаться с event delegation на реальных примерах
- Создать функцию для поиска элементов по сложным CSS-селекторам
- Изучить методы работы с формами через DOM

### Практическое задание:
Разработать интерактивную галерею изображений: создать HTML-структуру, реализовать навигацию между изображениями, добавить возможность отмечать избранные изображения с сохранением состояния в data-атрибутах, реализовать динамическое добавление и удаление изображений из галереи.
