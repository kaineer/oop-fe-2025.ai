# Задача урока 7: Модификация списка слов

## Контекст
В предыдущем уроке мы создали базовую структуру приложения для работы со списком слов кроссворда. Теперь необходимо добавить функционал для редактирования списка - удаления и обновления слов.

## Цель
Расширить функционал приложения, добавив возможность:
- Удалять слова из списка
- Добавлять новые слова через форму
- Обновлять существующие слова

## Требования

### Функциональные требования
1. **Удаление слов**: Пользователь должен иметь возможность удалить любое слово из списка
2. **Добавление слов**: Должна быть реализована форма для добавления новых слов
3. **Обновление слов**: При выборе существующего слова, форма должна переключаться в режим редактирования
4. **Валидация**: Проверка вводимых данных (непустые поля)

### Технические требования
1. Добавить методы `removeWord(word)` и `updateWord(word, description)` в класс `WordList`
2. Модифицировать функцию `renderWord()` для отображения кнопки удаления
3. Реализовать обработчик события `click` для удаления слов
4. Создать HTML-форму для добавления/редактирования слов
5. Реализовать логику переключения между режимами добавления и редактирования

### Архитектурные требования
- Сохранить существующую структуру классов `Word` и `WordList`
- Использовать делегирование событий для обработки кликов
- Обеспечить реактивное обновление интерфейса при изменении данных

## Ожидаемый результат
После выполнения урока приложение должно позволять:
- Просматривать список слов с описаниями
- Удалять слова по нажатию на кнопку "×"
- Добавлять новые слова через форму
- Редактировать существующие слова через ту же форму
- Сохранять изменения в интерфейсе без перезагрузки страницы

## Критерии успеха
- Все требования из урока 5 продолжают соблюдаться
- Интерфейс интуитивно понятен и отзывчив
- Код хорошо структурирован и следует принципам ООП
- Обработка ошибок и валидация данных реализованы корректно
