# Детальный план занятия: Модификация списка слов

## Метаданные урока
- **Номер урока**: 7
- **Тема**: Модификация списка, удаление элементов, форма для добавления слов
- **Тип**: Практическое занятие
- **Продолжительность**: 90 минут
- **Предварительные требования**: Уроки 1-6

## Цели обучения
- Научиться реализовывать операции удаления элементов из списка
- Освоить создание интерактивных форм для работы с данными
- Понять принципы делегирования событий в DOM
- Научиться переключать режимы работы интерфейса (добавление/редактирование)

## Структура занятия

### Введение (5 минут)
- Краткий обзор предыдущего урока (базовая структура приложения)
- Постановка задачи: необходимость редактирования списка слов
- Демонстрация ожидаемого результата

### Теоретическая часть (15 минут)

#### Принципы работы с DOM-событиями
- Делегирование событий: преимущества и применение
- Обработка кликов на динамически создаваемых элементах
- Работа с event.target для определения источника события

#### Архитектура приложения
- Расширение функциональности существующих классов
- Принцип единственной ответственности
- Реактивное обновление интерфейса

### Практическая часть (60 минут)

#### Шаг 1: Расширение класса WordList (10 минут)
- Добавление метода removeWord()
- Добавление метода updateWord()
- Валидация входных данных

#### Шаг 2: Модификация отображения (10 минут)
- Изменение шаблона для кнопки удаления
- Добавление CSS-стилей
- Обновление заголовков таблицы

#### Шаг 3: Реализация удаления слов (10 минут)
- Обработчик события click на списке
- Делегирование событий для кнопок удаления
- Интеграция с методом removeWord()

#### Шаг 4: Создание формы (10 минут)
- Добавление HTML-формы в index.html
- Настройка полей ввода и кнопки
- Базовые стили формы

#### Шаг 5: Добавление новых слов (10 минут)
- Обработчик события submit

- Интеграция с методом addWord()

#### Шаг 6: Реализация редактирования (10 минут)
- Логика выбора слова
- Визуальное выделение
- Обновление формы данными

### Заключение и тестирование (10 минут)
- Демонстрация готового приложения
- Тестирование всех сценариев использования
- Ответы на вопросы студентов

## Примечания для преподавателя

### Типичные ошибки студентов:
- Забывают про event.preventDefault() в обработчиках форм
- Неправильно определяют event.target при делегировании
- Не обеспечивают валидацию пользовательского ввода
- Забывают обновлять интерфейс после изменения данных

### Рекомендации по проведению:
- Делать акцент на принципах делегирования событий
- Показывать примеры хорошего и плохого UX
- Объяснять важность валидации данных
- Демонстрировать преимущества реактивного подхода
