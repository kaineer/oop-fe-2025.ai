# Детальный план занятия - Объекты в JavaScript

## Цели и задачи

### Основные цели:
- Познакомить студентов с различными способами создания объектов в JavaScript
- Научить работать со свойствами и методами объектов
- Объяснить концепции наследования и прототипирования
- Развить навыки использования классов и современных возможностей объектов

### Задачи обучения:
- Уметь объявлять объекты как литералы
- Понимать разницу между различными способами создания объектов
- Уметь создавать классы и работать с наследованием
- Понимать концепцию контекста `this` в методах объектов
- Уметь использовать геттеры и сеттеры

### Критерии успеха:
- Студент может самостоятельно создавать объекты разными способами
- Студент правильно использует методы объектов и понимает контекст `this`
- Студент создает классы с конструкторами и методами
- Студент применяет наследование между классами
- Студент использует геттеры и сеттеры для вычисляемых свойств

## Тайминг занятия (90 минут)

### Введение (15 минут)
- **0-5 мин:** Приветствие, организационные моменты, повторение предыдущей темы (функции)
- **5-10 мин:** Обзор темы занятия, постановка целей, важность объектов в программировании
- **10-15 мин:** Введение в объекты - что такое объекты и зачем они нужны, примеры из реальной жизни

### Основная часть (60 минут)

#### Блок 1: Создание объектов (20 минут)
- **15-25 мин:** Теория - литералы объектов, конструкторы, Object.create()
- **25-30 мин:** Практика - Задачи 1 и 2 (пустой объект, объект с методом)
- **30-35 мин:** Разбор отличий между способами создания, вопросы

#### Блок 2: Классы и методы (20 минут)
- **35-45 мин:** Теория - классы, конструкторы, методы, контекст `this`
- **45-50 мин:** Практика - Задачи 3, 4, 5 (класс Point с методами move и moveBy)
- **50-55 мин:** Разбор работы методов, вопросы по контексту `this`

#### Блок 3: Наследование и расширенные возможности (20 минут)
- **55-65 мин:** Теория - наследование, геттеры и сеттеры, прототипы
- **65-70 мин:** Практика - Задача 6 (метод mirror), дополнительные примеры
- **70-75 мин:** Разбор интересных фактов, вопросы по наследованию

### Заключение (15 минут)
- **75-80 мин:** Повторение ключевых концепций, интересные факты об объектах
- **80-85 мин:** Ответы на вопросы студентов, разбор сложных моментов
- **85-90 мин:** Резюме занятия, домашнее задание, подготовка к следующему уроку

## Критерии оценки

### Формативное оценивание:
- **Активность на занятии** - участие в обсуждениях, задавание вопросов
- **Выполнение практических заданий** - правильность синтаксиса, логика решения
- **Понимание концепций** - способность объяснить разницу между способами создания объектов
- **Работа в парах/группах** - совместное решение задач, помощь товарищам

### Суммативное оценивание:
- **Правильность выполнения задач** - соответствие кода требованиям задания
- **Понимание контекста выполнения** - корректное использование `this` в методах
- **Применение современных возможностей** - использование классов, наследования, геттеров/сеттеров
- **Читаемость кода** - понятные имена объектов и методов, правильное форматирование

### Контрольные точки:
- Задача 1: корректное создание пустого объекта
- Задача 2: создание объекта с методом, правильно использующим `this`
- Задача 3: создание класса Point с конструктором
- Задача 4: добавление метода move с правильной логикой
- Задача 5: добавление метода moveBy с правильной логикой смещения
- Задача 6: реализация метода mirror с правильной математической формулой

## Домашнее задание

### Обязательная часть:
1. Создать объект "книга" с свойствами (название, автор, год) и методом для вывода информации
2. Создать класс "прямоугольник" с полями ширина и высота, и методами для вычисления площади и периметра
3. Реализовать наследование: класс "животное" → класс "собака" с дополнительными методами
4. Создать объект с геттером, который вычисляет возраст на основе года рождения

### Дополнительная часть:
- Изучить разницу между `__proto__` и `prototype` в деталях
- Попрактиковаться с Object.create() для создания цепочек наследования
- Создать объект с несколькими уровнями наследования
- Изучить приватные поля и методы в классах (синтаксис #)

### Практическое задание:
Разработать систему для управления библиотекой с использованием классов: класс "Книга", класс "Читатель", класс "Библиотека" с методами для выдачи и приема книг.
