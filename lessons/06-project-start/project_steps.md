# Шаги проекта - Создание проекта кроссворда

## Обзор проекта

Проект направлен на создание редактора кроссворда с использованием Vanilla JavaScript и Vite. На текущем этапе студенты освоят создание проекта с нуля, добавление объектов для работы со словами и реализацию функций отрисовки данных на веб-странице.

## Ожидаемые результаты

По завершении проекта студенты должны продемонстрировать:

1. **Проект создается и отображается в браузере** - веб-приложение успешно запускается и отображает базовый интерфейс
2. **Создаются объекты для поддержки списка используемых слов** - реализованы классы Word и WordList для работы с данными
3. **Список используемых слов отрисовывается на странице** - данные корректно отображаются в пользовательском интерфейсе через функции отрисовки

## Шаг 0: Создание проекта

### Цели шага:
- Научить студентов создавать проект с помощью npm create
- Познакомить с базовой структурой Vite проекта
- Обеспечить правильную настройку окружения разработки

### Описание шага:
Студенты выполняют команду `npm create` для создания нового проекта. Выбирают опцию "Vanilla" и "JavaScript". После создания проекта устанавливают зависимости и запускают сервер разработки.

**Команды для выполнения:**
```bash
npm create
# Выбираем: Vanilla, JavaScript
npm install
npm run dev
```

**Действия:**
- Изменение заголовка документа в `index.html`
- Удаление лишних файлов из папок `public/` и `src/`
- Очистка основного JavaScript файла `main.js`
- Удаление стилей из `style.css`

### Ожидаемые результаты шага:
- Проект успешно создан и запускается на localhost
- Базовая структура файлов настроена
- Студенты понимают назначение основных файлов проекта
- **Достигнут ожидаемый результат: Проект создается и отображается в браузере**

## Шаг 1: Добавление объектов для поддержки списка используемых слов

### Цели шага:
- Научить студентов создавать классы для представления данных
- Познакомить с модульной организацией кода в JavaScript
- Обеспечить правильную структуру данных для работы со словами

### Описание шага:
Студенты создают классы `Word` и `WordList` для представления слов и управления списком слов. Организуют код в модульной структуре.

**Действия:**
- Создание файла `src/word.js` с классом `Word`
- Создание файла `src/word-list.js` с классом `WordList`
- Настройка импорта/экспорта между модулями
- Тестирование созданных классов в основном файле

### Ожидаемые результаты шага:
- Созданы классы `Word` и `WordList` с необходимыми свойствами и методами
- Модули корректно импортируются и экспортируются
- Студенты понимают принципы работы с классами и модулями в JavaScript
- **Достигнут ожидаемый результат: Создаются объекты для поддержки списка используемых слов**

## Шаг 2: Отрисовываем список используемых слов

### Цели шага:
- Научить студентов реализовывать функции отрисовки данных
- Познакомить с организацией кода для работы с пользовательским интерфейсом
- Обеспечить корректное отображение объектов на веб-странице

### Описание шага:
Студенты создают модуль с функциями отрисовки и интегрируют его с основным приложением. Реализуют отображение списка слов на странице.

**Действия:**
- Создание файла `src/renderers.js` с функциями отрисовки
- Реализация функции `renderWord` для отрисовки одного слова
- Реализация функции `renderWordList` для отрисовки всего списка
- Настройка импорта модулей в `main.js`
- Интеграция отрисовки в основной файл `main.js`

### Ожидаемые результаты шага:
- Созданы функции `renderWord` и `renderWordList` для работы с DOM
- Модуль отрисовки корректно интегрирован с основным приложением
- Список слов отображается на веб-странице
- Студенты понимают принципы разделения логики и представления
- **Достигнут ожидаемый результат: Список используемых слов отрисовывается на странице**

## Советы и рекомендации

### Для помощи студентам:
- **Начальные трудности**: Многие студенты могут испытывать проблемы с настройкой окружения. Рекомендуйте проверять версию Node.js и переустанавливать зависимости при необходимости.
- **Отладка модулей**: Научите студентов использовать консоль разработчика для проверки работы импорта/экспорта модулей.
- **DOM манипуляции**: Объясните принципы создания элементов и добавления их в DOM дерево.
- **Разделение ответственности**: Подчеркните важность разделения логики данных и логики отрисовки.

### Контрольные точки:
- **После шага 0**: Проверить, что проект запускается и отображает базовую страницу
- **После шага 1**: Убедиться, что созданы корректные классы и модули работают правильно
- **После шага 2**: Проверить отрисовку списка слов на странице
- Убедиться, что все файлы созданы и настроены корректно

### Решения для типичных проблем:
- **Ошибки при создании проекта**: Убедиться, что установлена последняя версия Node.js
- **Проблемы с модулями**: Проверить правильность путей импорта и синтаксис экспорта
- **Ошибки DOM**: Убедиться, что элементы созданы и добавлены в правильные контейнеры
- **Ошибки зависимостей**: Переустановить зависимости командой `npm install`
