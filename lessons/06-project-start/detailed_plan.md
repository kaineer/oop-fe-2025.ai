# Детальный план занятия - Создание проекта кроссворда

## Цели и задачи

### Основные цели:
- Научить студентов создавать веб-проект с нуля с использованием Vite
- Развить навыки работы с классами и объектами JavaScript
- Обеспечить понимание модульной организации кода
- Сформировать практические навыки реализации функций отрисовки данных
- Подготовить основу для дальнейшей разработки функциональности кроссворда

### Ожидаемые результаты:
После успешного выполнения проекта студенты должны продемонстрировать:

1. **Проект создается и отображается в браузере** - веб-приложение корректно запускается командой `npm run dev` и отображает базовый интерфейс в браузере
2. **Создаются объекты для поддержки списка используемых слов** - реализованы классы Word и WordList с необходимыми свойствами и методами для работы с данными
3. **Список используемых слов отрисовывается на странице** - данные из объектов корректно отображаются в пользовательском интерфейсе через специализированные функции отрисовки

### Задачи обучения:
- Уметь создавать проект с помощью npm create и настраивать Vite
- Создавать классы для представления данных с конструкторами и методами
- Организовывать код в модульной структуре с правильным импортом/экспортом
- Реализовывать функции отрисовки для работы с DOM элементами
- Интегрировать модули отрисовки с основным приложением
- Понимать принципы разделения логики данных и представления

### Критерии успеха:
- Студент самостоятельно создает и запускает проект
- Корректно создает классы Word и WordList с необходимыми свойствами и методами
- Организует код в модульной структуре с правильным импортом/экспортом
- Реализует функции renderWord и renderWordList для отрисовки данных
- Интегрирует модуль отрисовки с основным приложением
- Демонстрирует понимание принципов работы с классами, модулями и DOM
- Показывает умение отлаживать код и решать типичные проблемы

## Тайминг занятия (90 минут)

### Введение и подготовка (15 минут)
- **0-5 мин:** Приветствие, организационные моменты, обзор целей занятия и ожидаемых результатов
- **5-10 мин:** Объяснение проекта кроссворда, демонстрация ожидаемого результата с акцентом на три ключевых результата
- **10-15 мин:** Проверка наличия необходимого ПО (Node.js, редактор кода), ответы на вопросы по ожидаемым результатам

### Основная часть (65 минут)

#### Блок 1: Создание проекта (20 минут)
- **15-25 мин:** Теория - создание проекта с Vite, структура файлов, команды npm, ожидаемый результат "Проект создается и отображается в браузере"
- **25-35 мин:** Практика - выполнение команд npm create, установка зависимостей
- **35-40 мин:** Настройка базовой структуры - изменение заголовка, удаление лишних файлов
- **40-45 мин:** Очистка основного JavaScript файла main.js, удаление стилей из style.css, проверка запуска проекта

#### Блок 2: Создание объектов для слов (25 минут)
- **45-50 мин:** Теория - классы в JavaScript, свойства, методы, конструкторы, ожидаемый результат "Создаются объекты для поддержки списка используемых слов"
- **50-55 мин:** Практика - создание класса Word в файле src/word.js
- **55-60 мин:** Практика - создание класса WordList в файле src/word-list.js
- **60-65 мин:** Теория - модульная организация кода, импорт/экспорт
- **65-70 мин:** Практика - настройка импорта между модулями, создание тестовых данных

#### Блок 3: Отрисовка списка слов (20 минут)
- **70-75 мин:** Теория - принципы отрисовки данных, разделение логики и представления, ожидаемый результат "Список используемых слов отрисовывается на странице"
- **75-80 мин:** Практика - создание модуля renderers.js с функциями renderWord и renderWordList
- **80-85 мин:** Практика - настройка импорта модулей в main.js, интеграция отрисовки
- **85-90 мин:** Тестирование отрисовки списка слов, отладка возможных проблем

### Заключение (10 минут)
- **90-95 мин:** Демонстрация успешных проектов, проверка достижения всех трех ожидаемых результатов
- **95-100 мин:** Резюме занятия, домашнее задание, подготовка к следующему уроку

## Домашнее задание

### Обязательная часть:
1. Убедиться, что проект корректно запускается на домашнем компьютере и все три ожидаемых результата достигнуты
2. Добавить метод `removeWord` в класс WordList для удаления слов из списка по индексу
3. Реализовать метод `findWord` для поиска слов в списке по названию (регистронезависимый поиск)
4. Создать функцию `clearWordList` для очистки списка слов перед повторной отрисовкой
5. Добавить базовую стилизацию списка слов с помощью CSS (отступы, границы, шрифты)
6. Подготовить список вопросов по структуре проекта и принципам отрисовки для обсуждения

### Дополнительная часть:
- Добавить валидацию данных в конструкторы классов (проверка типов, длин строк)
- Реализовать метод `getWordCount` для получения количества слов в списке
- Создать простой интерфейс для добавления новых слов через HTML форму
- Изучить возможности статических методов и свойств в классах
- Реализовать сохранение списка слов в localStorage при закрытии страницы
- Добавить обработку ошибок в функциях отрисовки (проверка существования элементов)

### Практическое задание:
Подготовить проект к следующему занятию, обеспечив достижение всех ожидаемых результатов и:
- Чистую и понятную структуру файлов с комментариями
- Корректную работу всех команд npm и модулей
- Функциональные классы Word и WordList с дополнительными методами

- Отображение стилизованного списка слов на странице
- Готовность к интеграции с более сложным пользовательским интерфейсом
