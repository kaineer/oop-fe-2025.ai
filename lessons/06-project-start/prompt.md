# Промпт для создания документов по ролям

## Общий контекст урока:
- **Номер урока:** 6
- **Тема:** "Создаем проект, добавляем объекты, выводим список на страницу"
- **Продолжительность:** 90 минут
- **Тип:** Практическое занятие
- **Проект:** "crossword" - Редактирование и заполнение кроссворда
- **Ключевые концепции:** Создание проекта с Vite, настройка окружения, работа с HTML и JavaScript, создание объектов, работа с классами и модулями, отображение данных на странице
- **Цели обучения:** Умение создавать проект с нуля, настраивать окружение разработки, создавать объекты для представления данных, организовывать код в модули, реализовывать отображение данных на веб-странице

**Ожидаемые результаты:**
1. Проект создается и отображается в браузере
2. Создаются объекты для поддержки списка используемых слов
3. Список используемых слов отрисовывается на странице

**Этапы проекта (только описанные в meta.yaml):**
1. Создание проекта с помощью npm create
2. Настройка окружения разработки (Vite)
3. Изменение заголовка документа в index.html
4. Удаление лишних файлов из папок public/ и src/
5. Очистка основного JavaScript файла main.js от лишнего кода
6. Удаление стилей из style.css
7. Создание класса Word для представления слова
8. Создание класса WordList для управления списком слов
9. Создание функций отрисовки renderWord и renderWordList
10. Организация кода в модули
11. Отрисовка списка слов на странице

---

## Роль 1: task_description (Описание задачи)

**Аудитория:** Студенты
**Цель:** Подробное описание задачи для выполнения
**Тон:** Доступный, подробный
**Файл:** `task_description.md`

**Структура документа:**
1. **Описание проекта** - цели и задачи проекта кроссворда
2. **Ожидаемые результаты** - что должно быть достигнуто в результате
3. **Требования к решению** - что должно быть реализовано
4. **Элементы языка** - JavaScript конструкции, которые нужно использовать
5. **Этапы реализации** - пошаговое описание выполнения проекта

**Требования к содержанию:**
- Объяснять задачу так, как будто студент видит ее впервые
- Включать конкретные примеры ожидаемого результата
- Предоставлять четкие критерии успешного выполнения
- Использовать доступный язык, понятный новичкам
- Включать примеры кода для основных этапов
- **Включать только те этапы, которые описаны в meta.yaml**
- **Явно указывать ожидаемые результаты из meta.yaml**

---

## Роль 2: project_steps (Шаги проекта)

**Аудитория:** Преподаватель
**Цель:** Описание шагов проекта для выполнения на уроке
**Тон:** Кратко, общие детали
**Файл:** `project_steps.md`

**Структура документа:**
1. **Обзор проекта** - цели и ожидаемые результаты
2. **Цели каждого шага** - что студенты должны освоить на каждом этапе
3. **Описание каждого шага** - детали выполнения с командами и действиями
4. **Ожидаемые результаты каждого шага** - как проверить успешное выполнение
5. **Советы и рекомендации** - как помочь студентам при затруднениях

**Требования к содержанию:**
- Сосредоточиться на практических аспектах выполнения проекта
- Включить конкретные команды npm и шаги настройки
- Указать контрольные точки для проверки прогресса
- Предоставить решения для типичных проблем
- Включить ссылки на ресурсы внутри шагов с использованием формата filename@fragment-name
- **Включать только те шаги, которые явно описаны в meta.yaml**
- **Не включать гипотетические или предполагаемые шаги**
- **Связать каждый шаг с ожидаемыми результатами из meta.yaml**

---

## Роль 3: detailed_plan (Детальный план занятия)

**Аудитория:** Преподаватель/методист
**Цель:** Описание пройденных тем с почасовкой
**Тон:** Доступный, подробный
**Файл:** `detailed_plan.md`

**Структура документа:**
1. **Цели и задачи** - конкретные навыки, которые студенты приобретут
2. **Ожидаемые результаты** - что студенты должны уметь после занятия
3. **Тайминг занятия** - распределение 90 минут по этапам проекта
4. **Материалы и инструменты** - что понадобится для проведения занятия
5. **Критерии оценки** - как оценить успешное выполнение проекта
6. **Адаптация плана** - варианты для разных уровней студентов

**Требования к содержанию:**
- Предоставить детальный тайминг с разбивкой по 5-10 минутным интервалам
- Указать минимальные требования к оборудованию и ПО
- Определить критерии оценки каждого этапа проекта
- Включить резервные задания для быстрых студентов
- Предложить упрощенные варианты для студентов, требующих больше времени
- **Планировать время только для тех этапов, которые описаны в meta.yaml**
- **Явно указывать ожидаемые результаты из meta.yaml в разделе целей**

---

## Общие требования ко всем документам:
- Использовать только факты из метаданных урока и проекта
- Соответствовать тону и аудитории каждой роли
- Быть практико-ориентированными и инструктивными
- Включать конкретные примеры кода для проекта кроссворда
- Сосредоточиться на создании работающего прототипа
- Учитывать, что это первое практическое занятие по созданию проекта
- Использовать формат ссылок на ресурсы внутри шагов: filename@fragment-name
- При вставке кода в документы удалять комментарии REF и ENDREF
- **ВКЛЮЧАТЬ ТОЛЬКО ТЕ ШАГИ, КОТОРЫЕ ЯВНО ОПИСАНЫ В META.YAML**
- **НЕ ВКЛЮЧАТЬ ГИПОТЕТИЧЕСКИЕ ИЛИ ПРЕДПОЛАГАЕМЫЕ ШАГИ**
- **ЯВНО УКАЗЫВАТЬ ОЖИДАЕМЫЕ РЕЗУЛЬТАТЫ ИЗ META.YAML**

Создай три документа в папке `06-project-start` на основе этого промпта: `task_description.md`, `project_steps.md`, и `detailed_plan.md`.
