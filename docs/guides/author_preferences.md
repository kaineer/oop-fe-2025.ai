# Предпочтения автора по разработке

## Язык программирования для скриптов

### Предпочтение Python
В случае, если представляется выгодным написать скрипт для автоматизации действий, предпочитать скрипты на Python по следующим причинам:

1. **Универсальность окружения**
   - Окружения, в которых ведется разработка (в 99% случаев - Debian), имеют поддержку Python
   - Отсутствие зависимости от версии bash и флагов, с которыми bash был запущен

2. **Расширяемость и переиспользуемость**
   - Если функционал станет переиспользуемым, его можно будет вынести в отдельный модуль
   - В bash это тоже возможно, но не так удобно и структурированно

3. **Качество кода и поддерживаемость**
   - Более строгая типизация и структура
   - Лучшие инструменты для отладки и тестирования
   - Более читаемый и самодокументируемый код

### Компромиссы
Следующие аспекты Python могут быть менее удобными, но приемлемы:

- **Регулярные выражения** - поддержка выглядит более многословной, чем в других языках
- **Начальная сложность** - требует больше кода для простых задач по сравнению с bash

## Качество кода

### Проверка скриптов на Python
Все скрипты на Python должны проходить проверку на:

- **Отсутствие висящих импортов** - все импорты должны использоваться в коде
- **Соответствие PEP 8** - стиль кодирования
- **Обработку ошибок** - корректная обработка исключительных ситуаций

### Структура проектов
- Скрипты размещать в директории `scripts/`
- Скрипты, используемые в алиасах, использовать в `scripts/aliases`
- Сложные скрипты разделять на модули в `scripts/modules/`
- Документировать все скрипты в `scripts/README.md`

## Инструменты разработки

### Рекомендуемые инструменты
- **Для парсинга YAML/JSON** - использовать стандартные библиотеки Python
- **Для работы с файловой системой** - использовать `pathlib` вместо `os.path`
- **Для логирования** - использовать модуль `logging` вместо `print`

### Версионирование
- Указывать версию Python в shebang: `#!/usr/bin/env python3`
- Обеспечивать обратную совместимость при обновлении скриптов

## Миграция с bash
При замене bash-скриптов на Python:

1. Сохранять обратную совместимость интерфейса командной строки
2. Обновлять документацию с указанием новой команды
3. Помечать старые bash-скрипты как устаревшие
4. Удалять устаревшие скрипты после подтверждения работоспособности замены
